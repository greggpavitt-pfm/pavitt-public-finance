User-Administration-Guide

#This is a guide to the structure and administration of user profiles, permissions, granting and revoking of permissions.

#Guests
1) A Guest account is a temporary account that can access all features for a limited time period.
2) A Guest account must register their email address, and will be automatically approved.
3) After seven calendar days, the Guest account will automatically be suspended. The administrator, in the Administrator panel, will be able to renew the account for another seven days (there will be button for that). There will also be a configuration to cancel the account so that it can be re-registered under a standard account.

#Standard Accounts
1) A standard user will register for an account by prefilling in all the entries for their organization. There will also be an "N/A" option for an entry box.
2) Standard users must be approved by the administrator before using the features of the site. There will be checkboxes of approval for Training and for Practitioner, to allow separate approvals (both can be checked).
3) A standard user can be Suspended. The standard user can be reinstated (option in Administrative panel must be there), or permanently blacklisted (that option must be in the panel).


#Limts on Use
1) There will be a limits on use box for each user, one for training, and one for Practitioner. For training, it will be number of questions, and for practitioner number of submissions, which will be a monthly limit (7 days). There will also be a Default option. The organization (down to suborganization) to which the standard user is assigned will have a standard monthly limit, which will become the default. If no amount is selected for the standard user, then the default is used.
